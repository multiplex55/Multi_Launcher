### Highlights
- Adopted `eframe`/`egui_extras` 0.27.2 to pull in the latest egui fixes for widget sizing and theme stability ([release notes](https://github.com/emilk/egui/releases/tag/v0.27.2)).
- Updated `reqwest` 0.11.27 for the Tokio-based client bug fixes and HTTP/2 robustness improvements noted by upstream ([release notes](https://github.com/seanmonstar/reqwest/releases/tag/v0.11.27)).
- Bumped `winit` 0.29.15 and `windows` 0.58.0 to pick up Windows 11 input improvements and DPI fixes ([winit changelog](https://github.com/rust-windowing/winit/releases/tag/v0.29.15), [windows changelog](https://github.com/microsoft/windows-rs/releases/tag/v0.58.0)).
- Pulled in `notify` 6.1.1 and `chrono` 0.4.42 for filesystem watcher stability and timezone database refreshes ([notify release](https://github.com/notify-rs/notify/releases/tag/v6.1.1), [chrono release](https://github.com/chronotope/chrono/releases/tag/v0.4.42)).

### Git dependency review
- `rdev` remains pinned to commit `c14f2dc5` from upstream because it exposes the unreleased `0.6.0` API we depend on; the latest crates.io release (`0.5.3`) predates these changes and lacks the grab-mode fixes we require ([repository](https://github.com/Narsil/rdev/tree/c14f2dc5c8100a96c5d7e3013de59d6aa0b9eae2), [crates.io release](https://crates.io/crates/rdev/0.5.3)).
